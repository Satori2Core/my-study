# 【Agent 设计理念与业界参考】- Trae IDE 学习笔记

## 一、知识点拆解

### 1. Trae 设计 vs 通用 Agent 设计

**核心区别在于 Scope（范围）和 Implementation（实现）**：

| 维度 | Trae IDE 实现 | 通用 Agent 设计 |
|------|---------------|-----------------|
| **Scope** | Trae 特有的技能系统 | 业界通用的 Agent 架构模式 |
| **文件结构** | `.trae/skill/[name]/` 固定目录 | 无固定规范，灵活设计 |
| **触发机制** | 关键词匹配 + 场景触发 | 意图识别 + 工具调用链 |
| **配置方式** | 外部 JSON + Prompt 文件 | 动态配置 + 系统提示词 |
| **扩展性** | 基于目录的技能注册 | 插件化/API 驱动 |

**关系图**：

### 4. 业界规范与最佳实践

**Agent 设计的通用模式（源自 LLM Agent 研究）**：

| 组件 | 作用 | 实现方式 |
|------|------|----------|
| **角色定义** | 明确 Agent 身份 | System Prompt |
| **任务分解** | 复杂任务拆分子任务 | Chain of Thought |
| **工具调用** | 调用外部工具 | Function Calling |
| **状态管理** | 跟踪对话历史 | Memory Buffer |
| **反馈机制** | 优化行为 | Reinforcement Learning |

## 二、实战案例

### 案例1: 通用 Agent 架构实现

```python
class Agent:
    def __init__(self, system_prompt):
        self.system_prompt = system_prompt
        self.history = []
    
    def run(self, user_input):
        # 1. 构建完整的 prompt
        full_prompt = f"{self.system_prompt}\n\n"
        for msg in self.history:
            full_prompt += f"{msg['role']}: {msg['content']}\n"
        full_prompt += f"user: {user_input}"
        
        # 2. 调用 LLM
        response = llm.generate(full_prompt)
        
        # 3. 更新历史
        self.history.append({"role": "user", "content": user_input})
        self.history.append({"role": "assistant", "content": response})
        
        return response
```
