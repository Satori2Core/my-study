# 【Agent Rules 与 Skills 设计】- Trae IDE 学习笔记

## 一、知识点拆解

### 1. Skill（技能）的概念
- **定义**: Skill 是 Trae IDE 中可复用的功能模块，封装了特定业务逻辑
- **本质**: 一个 Skill = 触发规则 + 执行逻辑 + 输出格式
- **作用**: 让 Agent 能够执行特定任务（如代码审查、调试、生成代码等）

### 2. Agent Rules（代理规则）
- **定义**: 指导 Agent 行为的规则集合
- **组成**: 角色定义、执行规则、输出格式、禁止规则
- **作用**: 约束 Agent 的响应方式，确保输出符合预期

### 3. 核心文件结构
```
.trae/
└── skill/
    └── [skill-name]/
        ├── SKILL.md      # 技能元数据定义
        └── prompt.md     # Agent 提示词/规则
```

### 4. 触发机制
- **关键词触发**: 通过特定词汇激活 Skill
- **场景触发**: 根据用户意图自动匹配
- **配置驱动**: 读取外部配置文件动态调整行为

## 二、实战案例

### 案例1: 创建自定义 Skill 配置

**SKILL.md 文件结构**
```markdown
# 我的自定义技能

## 技能定义
- **名称**: my-skill
- **版本**: 1.0.0
- **描述**: 实现特定功能的技能

## 触发规则
- 关键词: ["关键词1", "关键词2"]

## 执行流程
用户提问 → 意图识别 → 执行逻辑 → 输出结果
```

**prompt.md 文件结构**
```markdown
# 我的 Agent 规则

## 核心角色
你是一个专业的助手，负责处理特定任务。

## 执行规则
1. 识别用户意图
2. 读取配置文件
3. 执行对应逻辑
4. 按规定格式输出

## 输出格式
```markdown
## 结果
- 内容1
- 内容2
```
```

## 三、关联知识点

### 前置知识
- **Markdown 语法**: 用于编写 SKILL.md 和 prompt.md
- **JSON 格式**: 用于配置文件
- **目录结构设计**: 理解 Trae IDE 的目录约定

### 后续知识
- **技能组合**: 多个 Skill 协作完成复杂任务
- **动态配置**: 根据环境变量调整行为
- **技能市场**: 共享和发现社区技能

### 交叉知识
- **LLM 提示工程**: prompt.md 本质是提示词工程
- **微服务架构**: Skill 类似独立服务
- **设计模式**: 策略模式（不同规则对应不同行为）
