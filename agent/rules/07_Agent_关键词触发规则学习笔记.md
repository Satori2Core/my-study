### 3. 关键词匹配的类型（认识阶段）

**简单理解**：关键词匹配就像"不同的查找方式" 🔍

| 匹配类型 | 生活化比喻 | 技术说明 | 适用场景 |
|----------|------------|----------|----------|
| **精确匹配** | 完全匹配 | 字符串完全相等 | 精确指令 |
| **模糊匹配** | 相似匹配 | 字符串相似度 | 容错处理 |
| **正则匹配** | 模式匹配 | 正则表达式匹配 | 复杂模式 |
| **语义匹配** | 意思匹配 | 语义相似度 | 智能理解 |

**详细说明**：

**① 精确匹配** 🎯
```
生活例子:
  你说"咖啡"，服务员知道你要咖啡
  你说"代码审查"，Agent 激活代码审查技能

技术实现:
  用户输入: "代码审查"
  关键词: "代码审查"
  匹配结果: 完全匹配 ✓

  用户输入: "帮我审查代码"
  关键词: "代码审查"
  匹配结果: 不匹配 ✗
```

**② 模糊匹配** 🔍
```
生活例子:
  你说"咖菲"，服务员知道你要咖啡（拼写错误）
  你说"代码审阅"，Agent 激活代码审查技能（同义词）

技术实现:
  用户输入: "代码审阅"
  关键词: "代码审查"
  相似度: 0.9 (高相似度)
  匹配结果: 匹配 ✓
```

**③ 正则匹配** 🎨
```
生活例子:
  你说"来杯咖啡"，服务员知道你要咖啡
  你说"学习Go语言"，Agent 激活学习教练技能

技术实现:
  用户输入: "学习Go语言"
  正则表达式: "学习.*语言"
  匹配结果: 匹配 ✓

  用户输入: "学习编程"
  正则表达式: "学习.*语言"
  匹配结果: 不匹配 ✗
```

**④ 语义匹配** 🧠
```
生活例子:
  你说"有点冷"，服务员知道你要热饮
  你说"代码有问题"，Agent 激活调试助手技能

  用户输入: "代码有问题"
  关键词: "代码审查"
```

**批注**：不同的匹配类型就像不同的"查找方式"，有的精确（精确匹配），有的容错（模糊匹配），有的灵活（正则匹配），有的智能（语义匹配）。实际应用中，通常会组合使用多种匹配类型。

### 案例1: 精确匹配实现

**场景**：用户询问代码审查

**生活化例子**：
```
用户: "代码审查"
  ↓
Agent 检测到"代码审查"关键词
  ↓
就像服务员听到"咖啡"就知道要准备咖啡
  ↓
Agent 激活代码审查技能
```

**技术实现**：
```python
def exact_matching(user_input, keywords):
    """
    精确匹配函数

    参数:
        user_input: 用户输入的文本
        keywords: 关键词列表

    返回:
        匹配的关键词列表

    算法原理:
        1. 遍历每个关键词
        2. 检查用户输入是否完全包含关键词
        3. 收集所有匹配的关键词

    时间复杂度: O(n * m)，其中 n 是输入长度，m 是关键词数量
    空间复杂度: O(k)，其中 k 是匹配的关键词数量
    """
    matched_keywords = []

    for keyword in keywords:
        # 检查用户输入是否完全包含关键词
        if keyword in user_input:
            matched_keywords.append(keyword)

    return matched_keywords

# 测试用例
```

### 案例2: 模糊匹配实现

**场景**：用户输入有拼写错误或同义词

**生活化例子**：
```
用户: "代码审阅"（同义词）
  ↓
Agent 检测到"代码审阅"与"代码审查"相似
  ↓
就像服务员听到"咖菲"知道你要咖啡
  ↓
Agent 激活代码审查技能
```

**技术实现**：
```python
from difflib import SequenceMatcher

def fuzzy_matching(user_input, keywords, threshold=0.7):
    """
    模糊匹配函数

    参数:
        user_input: 用户输入的文本
        keywords: 关键词列表
        threshold: 相似度阈值（0-1）

    返回:
        匹配的关键词列表和相似度

    算法原理:
        1. 遍历每个关键词
        2. 计算用户输入与关键词的相似度
        3. 收集相似度超过阈值的关键词

    时间复杂度: O(n * m * k)，其中 n 是输入长度，m 是关键词数量，k 是最长字符串长度
    空间复杂度: O(k)，其中 k 是匹配的关键词数量
    """
    matched_keywords = []

    for keyword in keywords:
        # 计算相似度
        similarity = SequenceMatcher(None, user_input, keyword).ratio()

        # 检查是否超过阈值
        if similarity >= threshold:
            matched_keywords.append({
                "keyword": keyword,
                "similarity": similarity
            })

    # 按相似度排序
    matched_keywords.sort(key=lambda x: x["similarity"], reverse=True)

    return matched_keywords

# 测试用例
test_cases = [
    ("代码审阅", ["代码审查", "review"], 0.7),
    ("代码审阅", ["代码审查", "review"], 0.9),
    ("revieew", ["代码审查", "review"], 0.7),
    ("检查代码", ["代码审查", "review"], 0.7),
]

for user_input, keywords, threshold in test_cases:
    result = fuzzy_matching(user_input, keywords, threshold)
    print(f"输入: {user_input}")
    print(f"关键词: {keywords}")
    print(f"阈值: {threshold}")
    print(f"匹配结果: {result}")
    print("-" * 50)
```

**输出结果**：
```
输入: 代码审阅
关键词: ['代码审查', 'review']
阈值: 0.7
匹配结果: [{'keyword': '代码审查', 'similarity': 0.8571428571428571}]
--------------------------------------------------
输入: 代码审阅
关键词: ['代码审查', 'review']
阈值: 0.9
匹配结果: []
--------------------------------------------------
输入: revieew
关键词: ['代码审查', 'review']
阈值: 0.7
匹配结果: [{'keyword': 'review', 'similarity': 0.8571428571428571}]
--------------------------------------------------
输入: 检查代码
关键词: ['代码审查', 'review']
阈值: 0.7
匹配结果: []
--------------------------------------------------
```

**批注**：模糊匹配就像"容错处理"，它可以处理用户的拼写错误、同义词等情况。它的优点是容错性强，但缺点是计算复杂度高，需要设置合适的阈值。

### 案例3: 正则匹配实现

**场景**：用户输入符合特定模式

**生活化例子**：
```
用户: "学习Go语言"
  ↓
Agent 检测到符合"学习.*语言"模式
  ↓
就像服务员听到"来杯咖啡"就知道要咖啡
  ↓
Agent 激活学习教练技能
```

**技术实现**：
```python
import re

def regex_matching(user_input, patterns):
    """
    正则匹配函数

    参数:
        user_input: 用户输入的文本
        patterns: 正则表达式列表

    返回:
        匹配的模式列表和匹配结果

    算法原理:
        1. 遍历每个正则表达式
        2. 检查用户输入是否匹配正则表达式
        3. 收集所有匹配的模式

    时间复杂度: O(n * m)，其中 n 是输入长度，m 是正则表达式数量
    空间复杂度: O(k)，其中 k 是匹配的模式数量
    """
    matched_patterns = []

    for pattern in patterns:
        # 编译正则表达式
        regex = re.compile(pattern)

        # 检查是否匹配
        match = regex.search(user_input)
        if match:
            matched_patterns.append({
                "pattern": pattern,
                "match": match.group()
            })

    return matched_patterns

# 测试用例
test_cases = [
    ("学习Go语言", [r"学习.*语言"]),
    ("学习编程", [r"学习.*语言"]),
    ("学习Java语言", [r"学习.*语言"]),
    ("学习", [r"学习.*语言"]),
]

for user_input, patterns in test_cases:
    result = regex_matching(user_input, patterns)
    print(f"输入: {user_input}")
    print(f"正则表达式: {patterns}")
    print(f"匹配结果: {result}")
    print("-" * 50)
```

**输出结果**：
```
输入: 学习Go语言
正则表达式: ['学习.*语言']
匹配结果: [{'pattern': '学习.*语言', 'match': '学习Go语言'}]
--------------------------------------------------
输入: 学习编程
正则表达式: ['学习.*语言']
匹配结果: []
--------------------------------------------------
输入: 学习Java语言
正则表达式: ['学习.*语言']
匹配结果: [{'pattern': '学习.*语言', 'match': '学习Java语言'}]
--------------------------------------------------
输入: 学习
正则表达式: ['学习.*语言']
匹配结果: []
--------------------------------------------------
```

**批注**：正则匹配就像"模式匹配"，它可以匹配符合特定模式的文本。它的优点是灵活强大，可以匹配复杂的模式，但缺点是正则表达式复杂，学习成本高。

**验证示例**：
```python
import time
import random

def performance_test(matching_function, user_inputs, keywords):
    """
    性能测试函数

    参数:
        matching_function: 匹配函数
        user_inputs: 用户输入列表
        keywords: 关键词列表

    返回:
        性能测试结果
    """
    # 记录开始时间
    start_time = time.time()

    # 执行匹配
    for user_input in user_inputs:
        matching_function(user_input, keywords)

    # 记录结束时间
    end_time = time.time()

    # 计算执行时间
    execution_time = end_time - start_time

    # 计算平均时间
    avg_time = execution_time / len(user_inputs)

    return {
        "total_time": execution_time,
        "avg_time": avg_time,
        "input_count": len(user_inputs),
        "keyword_count": len(keywords)
    }

# 生成测试数据
user_inputs = ["代码审查" for _ in range(1000)]
keywords = ["代码审查", "review", "检查代码"]

# 执行性能测试
result = performance_test(exact_matching, user_inputs, keywords)
```

## 四、底层原理（原理阶段）

### 1. 字符串匹配算法

**算法原理**：
```python
def naive_string_matching(text, pattern):
    """
    朴素字符串匹配算法

    算法步骤:
        1. 遍历文本的每个位置
        2. 在每个位置检查是否匹配模式
        3. 如果匹配，记录匹配位置

    时间复杂度: O(n * m)，其中 n 是文本长度，m 是模式长度
    空间复杂度: O(1)，不需要额外空间

    批注: 这是最简单的字符串匹配算法，虽然效率不高，但易于理解和实现。
    """
    n = len(text)
    m = len(pattern)
    matches = []

    for i in range(n - m + 1):
        # 检查从位置 i 开始是否匹配
        match = True
        for j in range(m):
            if text[i + j] != pattern[j]:
                match = False
                break

        if match:
            matches.append(i)

    return matches
```

**批注**：朴素字符串匹配算法就像"逐个字符比较"，虽然简单，但效率不高。在实际应用中，通常会使用更高效的算法，如 KMP 算法、Boyer-Moore 算法等。

### 2. 相似度计算算法

**算法原理**：
```python
def levenshtein_distance(s1, s2):
    """
    Levenshtein 距离算法（编辑距离）

    算法步骤:
        1. 创建一个二维数组 dp
        2. dp[i][j] 表示 s1[0:i] 和 s2[0:j] 的编辑距离
        3. 动态规划计算编辑距离

    时间复杂度: O(n * m)，其中 n 和 m 是两个字符串的长度
    空间复杂度: O(n * m)，需要二维数组

    批注: Levenshtein 距离是衡量两个字符串相似度的重要指标，它表示将一个字符串转换成另一个字符串所需的最少编辑操作次数。
    """
    n, m = len(s1), len(s2)

    # 创建二维数组
    dp = [[0] * (m + 1) for _ in range(n + 1)]

    # 初始化边界条件
    for i in range(n + 1):
        dp[i][0] = i
    for j in range(m + 1):
        dp[0][j] = j

    # 动态规划计算
    for i in range(1, n + 1):
        for j in range(1, m + 1):
            if s1[i - 1] == s2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                dp[i][j] = min(
                    dp[i - 1][j] + 1,      # 删除
                    dp[i][j - 1] + 1,      # 插入
                    dp[i - 1][j - 1] + 1   # 替换
                )

    return dp[n][m]

def similarity_score(s1, s2):
    """
    计算相似度分数

    算法原理:
        1. 计算 Levenshtein 距离
        2. 根据距离计算相似度分数

    相似度分数 = 1 - (编辑距离 / 最大字符串长度)

    批注: 相似度分数的范围是 [0, 1]，值越大表示两个字符串越相似。
    """
    distance = levenshtein_distance(s1, s2)
    max_length = max(len(s1), len(s2))
    similarity = 1 - (distance / max_length) if max_length > 0 else 1
    return similarity
```

**批注**：Levenshtein 距离算法就像"计算编辑成本"，它计算将一个字符串转换成另一个字符串所需的最少编辑操作次数。相似度分数则是基于编辑距离计算的，值越大表示两个字符串越相似。
